Privacy Policy

Effective date: March 9, 2026

Kuma Uptime Manager ("the App") respects your privacy. This Privacy Policy explains how the App handles information when you use it.

1. Information You Provide

To connect to your Uptime Kuma server, you may provide configuration details such as a server URL, API keys, or optional management credentials. These details are used only to authenticate and communicate with your Uptime Kuma instance.

2. Local Storage

Sensitive credentials such as API keys and login credentials are stored securely using the macOS Keychain. Non-sensitive preferences, such as user interface settings and polling configuration, may be stored locally using standard macOS application preferences.

3. Network Communication

The App communicates only with the Uptime Kuma server that you configure. No information is sent to the developer or to any external analytics or advertising services.

4. Data Collection

Kuma Uptime Manager does not collect, store, or transmit personal data to the developer. All monitoring information displayed in the App is retrieved directly from the Uptime Kuma server configured by the user.

5. Notifications

If enabled, the App may use macOS system notifications to alert you when monitor states change, such as when a service goes offline or recovers. These notifications are generated locally on your device.

6. Data Sharing

The App does not sell, rent, or share user data with third parties.

7. Data Retention

Any configuration data you provide remains on your device until it is manually removed, the application settings are reset, or the App is uninstalled.

8. Third-Party Services

The App interacts with user-configured Uptime Kuma servers. The handling of monitoring data by those servers is governed by the policies of the server operator.

9. Changes to This Policy

This Privacy Policy may be updated from time to time. Any changes will be reflected on this page with an updated effective date.

10. Contact

If you have questions about this Privacy Policy, you can contact the developer through the App Store support link or via the project repository on GitHub.
